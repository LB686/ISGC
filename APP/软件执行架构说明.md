# 启发一体控制器软件执行架构说明

> **版本**: V2.0  
> **日期**: 2026-04-27  
> **适用硬件**: STM32F405RGT6, 168MHz, 无RTOS

---

## 一、总体架构

系统采用**裸机前后台模型**：
- **实时任务**由硬件中断驱动（20kHz FOC + 1kHz 状态机）
- **非实时任务**在主循环中轮询执行
- **串口协议解析**采用中断回调方式，无需轮询FIFO

```
┌─────────────────────────────────────────────────────────────┐
│                        中断层 (前台)                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │ TIM1_BRK │  │ ADC_IRQn │  │ TIM5_IRQn│  │ USART3   │    │
│  │ 优先级0  │  │ 优先级1  │  │ 优先级2  │  │ 优先级4  │    │
│  │ 硬件封波 │  │ 20kHz FOC│  │ 1ms时基  │  │ 字节回调 │    │
│  │ (INA303) │  │ 电流环   │  │ 状态机   │  │ 协议解析 │    │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      主循环层 (后台)                          │
│                                                             │
│   while (1) {                                               │
│       hal_fast_loop();          // 串口FIFO+发送轮询        │
│       while(时间到10ms) hal_100hz_loop();   // 按键扫描    │
│       while(时间到20ms) hal_50hz_loop();    // 后台任务    │
│       while(时间到500ms)hal_2hz_loop();     // LED+喂狗    │
│       ...                                                   │
│   }                                                         │
│                                                             │
│   注意：1000Hz任务已移至TIM5中断，主循环中不再轮询         │
└─────────────────────────────────────────────────────────────┘
```

## 二、时钟与时间管理

### 2.1 TIM5 — 系统时基

| 参数 | 值 | 说明 |
|------|-----|------|
| 时钟源 | APB1 = 84MHz | |
| 预分频 | 83 | 计数频率 = 84MHz / 84 = 1MHz |
| ARR | 999 | 中断周期 = 1000 / 1MHz = 1ms |
| 中断优先级 | 2 | 低于ADC(1)，高于通信(3/4) |

**ISR执行内容**（`HAL_TIM_PeriodElapsedCallback`，在 `main.c` 中）：
```c
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim->Instance == TIM5) {
        system_timer_tick();        /* 1ms计时 */
        hal_1000hz_loop();          /* 1kHz实时任务：INA303采样 + 电机状态机 */
    }
}
```

> **关键设计**：`hal_1000hz_loop()` **仅在TIM5中断中执行一次**，主循环中不再轮询，彻底避免重入。

### 2.2 时间获取

`bsp_system_time.c` 提供以下接口：

| 函数 | 精度 | 用途 |
|------|------|------|
| `get_system_time_ms()` | 1ms | 主循环时间轮询 |
| `get_system_time_us()` | ~1us | 基于timer_tick_count × 1000 + TIM5_CNT |
| `delay_ms()` / `delay_us()` | 忙等待 | 初始化阶段延时，**运行时慎用** |

> ⚠️ **已知限制**：`timer_tick_count` 为 `uint64_t`，在 32 位 MCU 上读取非原子，建议在 `get_system_time_us()` 中加关中断保护。

## 三、中断执行流程

### 3.1 ADC1/2 全局中断 (20kHz, 优先级1)

**触发源**: TIM1_CH4 OC4REF 上升沿 → ADC1/ADC2 注入组同步触发

**执行流程**（在 `ADC_IRQHandler` 中）：

```
TIM1_CC4 上升沿
    │
    ▼
ADC1 注入组转换 (Iu→Iv→Iw, 3通道, ~2.14μs)
ADC2 注入组转换 (BEMF_A→BEMF_B→BEMF_C, 3通道, ~2.14μs)
    │
    ▼
JEOC中断 (ADC_IRQn, 优先级1)
    ├── 读取 ADC1->JDR1/2/3 (Iu, Iv, Iw)
    ├── 读取 ADC2->JDR1/2/3 (BEMF_A, BEMF_B, BEMF_C)
    ├── 调用 MOTOR_FastControlTask()
    └── 调用 HAL_ADC_IRQHandler(&hadc1/2/3) —— 处理其他ADC标志
            │
            └── MOTOR_FastControlTask()
                    │
                    ├── 读取电流 (BSP_INA303_GetCurrentX)
                    ├── 20kHz软件过流检查 (动态阈值)
                    ├── 电角度积分
                    ├── MOTOR_Clarke() → MOTOR_Park() → PI → MOTOR_InvPark()
                    ├── MOTOR_SVM() → SVPWM
                    └── 更新 TIM1->CCR1/2/3
```

**执行时间预算**：< 25μs (50μs周期的50%)

### 3.2 TIM5 全局中断 (1ms, 优先级2)

**触发源**: TIM5 更新事件 (ARR=999)

**执行内容**：
```c
void HAL_TIM_PeriodElapsedCallback(TIM5) {
    system_timer_tick();            // timer_tick_count++
    hal_1000hz_loop();              // 1kHz实时任务（见下方详情）
}
```

**`hal_1000hz_loop()` 执行顺序**（在 `bsp.c` 中）：
```c
void hal_1000hz_loop(void) {
    BSP_INA303_FastTask();          /* 步骤1：INA303滤波 + ADC3轮询(Vbus/Temp) */
    MOTOR_SlowTask_1kHz();          /* 步骤2：状态机 + 故障检测（使用步骤1的最新数据） */
}
```

> **设计要点**：`BSP_INA303_FastTask()` 必须先执行，确保 `MOTOR_SlowTask_1kHz()` 中的母线电压、温度、电流滤波值是最新的。

### 3.3 TIM1 刹车中断 (硬件过流, 优先级0)

**触发源**: INA303 ALERT1 过流信号 → 逻辑门 → TIM1_BKIN

**执行内容**:
```c
void TIM1_BRK_TIM9_IRQHandler(void) {
    HAL_TIM_IRQHandler(&htim1);     // HAL自动关闭PWM输出
}
```

**响应时间**: < 1μs (硬件级封波)

### 3.4 USART3 中断 (优先级4)

**触发源**: 接收到1字节数据

**执行内容**（在 `HAL_UART_RxCpltCallback` 中）：
```c
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart) {
    if (huart == &huart3) {
        usart3_got_byte(usart3_data);       /* 回调模式：直接给上层；非回调模式：压入FIFO */
        HAL_UART_Receive_IT(&huart3, &usart3_data, 1);
    }
}
```

**设计**：采用**回调与FIFO互斥**机制：
- **注册了回调**（如 `APP_MotorUartParse_Init()`）：字节直接传给 `APP_MotorUartParse_Process()`，**不进FIFO**
- **未注册回调**：字节压入FIFO，由 `hal_fast_loop()` → `usart_dataparse()` 轮询消费

## 四、主循环执行流程

### 4.1 主循环架构 (main.c)

```c
while (1) {
    hal_fast_loop();          // 空闲轮询，每次循环都执行

    // 时间分频轮询（使用while防止跳过tick）
    while (get_system_time_ms() >= last_100Hz_timestamp + 10) {
        last_100Hz_timestamp += 10;
        hal_100hz_loop();     // 100Hz
    }
    while (get_system_time_ms() >= last_50Hz_timestamp + 20) {
        last_50Hz_timestamp += 20;
        hal_50hz_loop();      // 50Hz
    }
    while (get_system_time_ms() >= last_2Hz_timestamp + 500) {
        last_2Hz_timestamp += 500;
        hal_2hz_loop();       // 2Hz
    }
    /* ... 1Hz, 10Hz ... */
    /* 注意：1000Hz任务已移至TIM5中断，此处不再轮询 */
}
```

### 4.2 各回调函数职责

| 函数 | 频率 | 执行位置 | 职责 |
|------|------|---------|------|
| `hal_fast_loop()` | 尽可能快 | `bsp.c` | 串口FIFO轮询 (`usart_dataparse()`) + 发送状态检查 (`read_usart_state()`) |
| `hal_1000hz_loop()` | 1ms | **`bsp.c`，但**由TIM5中断调用 | `BSP_INA303_FastTask()` → `MOTOR_SlowTask_1kHz()` |
| `hal_100hz_loop()` | 10ms | `bsp.c` | 按键扫描 (`bsp_KeyScan10ms()`) |
| `hal_50hz_loop()` | 20ms | `bsp.c` | 后台非实时任务 (`MOTOR_BackgroundTask()`) |
| `hal_10hz_loop()` | 100ms | `bsp.c` | (预留) |
| `hal_2hz_loop()` | 500ms | `bsp.c` | LED翻转 + 看门狗喂狗 (`HAL_IWDG_Refresh`) |
| `hal_1hz_loop()` | 1s | `bsp.c` | (预留) |

## 五、状态机执行时序

### 5.1 主状态机状态转换

```
上电复位
    │
    ▼
[POWER_ON] ──100ms延时──→ [HW_CHECK]
                              │
         ┌────────────────────┘
         ↓ 自检通过(Vbus∈[22,35]V, Temp<80°C)
    [STANDBY] ←────────────────────┐
         │                          │
         ↓ 收到启动指令              │
    [STARTUP_DRAG]                  │
         │                          │
         ├── 子阶段1: VF_OPEN (0~5ms)
         ├── 子阶段2: CURR_TAKEOVER (5~50ms)
         ├── 子阶段3: IF_STEADY (50ms~4s)
         └── 子阶段4: BEMF_CHECK (4s后)
              │
              ├── 成功 ──→ [GENERATING]
              └── 失败 ──→ [SAFE_STOP] ──→ [STANDBY]
                                   │
                              故障未清除 ──→ [FAULT]
```

### 5.2 启动阶段四段式时序

| 子阶段 | 时间窗口 | 角度来源 | 电流控制 | 频率 | OC阈值 |
|--------|---------|---------|---------|------|--------|
| VF_OPEN | 0~5ms | 虚拟角度 | 开环Vq=0.9 | 200Hz | 400A |
| CURR_TAKEOVER | 5~50ms | 虚拟角度 | 电流环PI | 斜坡 | 150A |
| IF_STEADY | 50ms~4s | 虚拟角度 | Iq=40A | 0→300Hz斜坡 | 65A |
| BEMF_CHECK | 4s后 | 冻结 | 停止 | 0 | - |

### 5.3 20kHz中断 vs 1kHz任务的职责边界

| 职责 | 执行环境 | 频率 | 说明 |
|------|---------|------|------|
| 电流采样 | ADC ISR | 20kHz | 读INA303 → JDR寄存器 |
| FOC数学 | ADC ISR | 20kHz | MOTOR_Clarke→Park→PI→InvPark→SVM |
| PWM更新 | ADC ISR | 20kHz | 写TIM1_CCR |
| 软件过流 | ADC ISR | 20kHz | 直接比较，超限立即封波 |
| INA303采样/滤波 | TIM5 ISR | 1kHz | ADC3轮询(Vbus/Temp) + 滑动平均 |
| 状态切换 | TIM5 ISR | 1kHz | 阶段一→二→三→四 |
| 频率斜坡 | TIM5 ISR | 1kHz | 电频率0→300Hz渐变 |
| 电流指令 | TIM5 ISR | 1kHz | Iq指令生成 |
| PI积分预置 | TIM5 ISR | 1kHz | 阶段切换时无扰衔接 |
| 故障检测 | TIM5 ISR | 1kHz | 过压/欠压/过温/超时 |

## 七、VESC数学库移植说明

`APP/app_motor_math.c/h` 中的核心算法已与 VESC 源码保持一致：

| 本工程函数 | VESC对应函数 | 状态 |
|-----------|-------------|------|
| `MOTOR_SincosBetter` | `utils_fast_sincos_better` | ✅ 一致 |
| `MOTOR_Atan2` | `utils_fast_atan2` | ✅ 一致 |
| `MOTOR_SVM` | `foc_svm` | ✅ 一致 |
| `MOTOR_Clarke` | 简化Clark (`i_alpha=ia, i_beta=ONE_BY_SQRT3*ia+TWO_BY_SQRT3*ib`) | ✅ 一致 |
| `MOTOR_Park` | Park变换 (`id=c*iα+s*iβ, iq=c*iβ-s*iα`) | ✅ 一致 |
| `MOTOR_InvPark` | InvPark变换 | ✅ 一致 |

**新增移植的工具函数**（从VESC `utils.c/h` 移植，统一 `MOTOR_` 前缀）：
- `MOTOR_StepTowards()` —— 斜坡限幅
- `MOTOR_AngleDifferenceRad()` —— 角度差计算
- `MOTOR_Deadband()` —— 死区处理
- `MOTOR_MiddleOf3()` / `MOTOR_MiddleOf3Int()` —— 三取中滤波
- `MOTOR_MinAbs()` / `MOTOR_MaxAbs()` —— 绝对值比较
- `MOTOR_Map()` —— 线性映射
- `MOTOR_SaturateVector2D()` —— 二维向量圆限幅

> `MOTOR/` 目录下的 VESC 源码可安全删除，所有需要使用的数学函数已提取到 `APP/app_motor_math.c/h`。

## 八、关键配置检查表（上电前必查）

| # | 检查项 | 正确值 | 错误后果 |
|---|--------|--------|---------|
| 1 | TIM1 CH4 Pulse | 4050 | CCR4=4199→采样窗口不够→电流数据错误 |
| 2 | ADC触发沿 | **RISING** | FALLING→下桥臂即将关闭时触发→转换被打断 |
| 3 | 注入组采样时间 | **3 Cycles** | 15 cycles→3.86μs > 2.5μs窗口→转换中断 |
| 4 | INA303 REF电压 | **1.6V** | 1.65V→零偏误差62A |
| 5 | TIM5 ARR | 999 | 1kHz时基 |
| 6 | NVIC优先级分组 | Group 4 | 全抢占优先级 |
